# Lab Setup

These labs are designed to be run inside a virtual machine (VM) created using Vagrant and KVM/libvirt. This ensures a consistent environment for all users and avoids the need to install dependencies directly on your host system.

## Install KVM Hypervisor

You want to install vagrant and the libvirt provider on a preferably Debian-based Linux host hypervisor system:

```bash
sudo apt-get install -y git vagrant vagrant-libvirt \
  libvirt-daemon-system libvirt-daemon-driver-qemu libvirt-clients virtinst virt-manager \
  qemu-kvm qemu-utils
```

If on other, non-debian package manager, based systems, please refer to the respective package manager documentation and repository information for installing analogs of these packages. And yes you could probably get by using other hypervisors supported by Vagrant such as VirtualBox even on other operating systems, but we strongly recommend using Linux.

## Clone and build the brothaman-z12-box

A one time build of the brothaman/z12 (Debian 12 -- Bookworm with ZFS) vagrant box is required before first use:

```shell
git clone https://github.com/brothaman-org/brothaman-z12-box
cd brothaman-z12-box
./build.sh
```

### Session Libvirt Preflight (fix for `qemu:///session` socket errors)

If `./build.sh` fails with an error like:

```bash
Error making a connection to libvirt URI qemu:///session:
... /run/user/<uid>/libvirt/libvirt-sock: No such file or directory
```

your per-user libvirt session is not active (or your shell lacks a valid user runtime environment).

Run this before `./build.sh`:

```bash
# confirm you are in a real user session (not a root shell)
id -u
printf 'XDG_RUNTIME_DIR=%s\n' "${XDG_RUNTIME_DIR:-<unset>}"

# ensure user session services are available
systemctl --user daemon-reload || true
systemctl --user start dbus.service dbus.socket 2>/dev/null || true
systemctl --user start libvirtd.service 2>/dev/null || true

# verify session libvirt
virsh -c qemu:///session uri
```

If your distro does not provide user-scope libvirt services, use system-scope libvirt instead:

```bash
sudo systemctl enable --now libvirtd
export LIBVIRT_DEFAULT_URI=qemu:///system
virsh -c qemu:///system uri
./build.sh
```

`[fog][WARNING] Unrecognized arguments: libvirt_ip_command` is a compatibility warning from ruby-fog/vagrant-libvirt and is usually non-fatal; the hard failure is the missing libvirt socket.


## Clone and fire up the brothaman lab VM

Now we clone the brothaman repo and fire up the brothaman lab VM:

```shell
git clone https://github.com/brothaman-org/brothaman
cd brothaman
vagrant up && vagrant snapshot save before-starting
```
